## Важные имена маршрутов (Django)

Чтобы не сломать ссылки, пользуемся существующими именами:

- `frontpage` — главная `/`
- `catalog:index` — каталог `/catalog/`
- `catalog:category` slug — категория, например `/catalog/8-marta/`
- `cart:cart` — корзина `/cart/`
- `staticpages:staticpage` — статичная страница по `url`, например `url='clients'` → `/clients/`
- `contacts-page` — контакты `/contacts/`
- `crm:feedback-message-request` — форма обратной связи

Новые URL и view не добавляем без необходимости — только подстановка в уже существующие шаблоны хедера/футера/главной.
